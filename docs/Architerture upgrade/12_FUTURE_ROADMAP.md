# Future Roadmap

## Future themes
- better document understanding
- stronger hybrid retrieval
- graph-first reasoning
- memory policy
- prompt optimization
- observability
- selective infrastructure specialization

## What to keep custom
- Recall knowledge model
- memory policy
- graph semantics
- branching semantics
- retrieval routing policy
- security policy
- analytics schema
- logging event schema

## What to adopt only when needed
- Unstructured
- Instructor or Outlines
- DSPy
- LlamaIndex
- Haystack
- Mem0
- LangGraph
- Qdrant
- Neo4j
- OpenTelemetry

## Guiding rule
Adopt a tool only if it reduces actual complexity or raises measurable quality.

## Vision
The future Recall should not just retrieve documents. It should:
- surface related ideas
- remember useful facts
- expose knowledge neighborhoods
- detect patterns and contradictions
- stay understandable and maintainable
